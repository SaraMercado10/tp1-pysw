# 🌍 Proyecto Web de Turismo

Sitio web desarrollado como práctica integradora utilizando **HTML5 y CSS3**, enfocado en la creación de interfaces modernas, responsivas y accesibles sin el uso de frameworks.

## 🎯 Objetivo del Proyecto

El objetivo de este trabajo es aplicar conceptos fundamentales y avanzados de desarrollo web, incluyendo:

- Estructuración semántica con HTML5  
- Diseño visual con CSS3  
- Diseño responsivo (Responsive Design)  
- Animaciones y efectos avanzados  
- Buenas prácticas de organización de código  
- Accesibilidad web  

---

## 🧱 Estructura del Sitio

El sitio está compuesto por múltiples páginas interconectadas:

- index.html → Página principal (Home)  
- destinos.html → Destinos turísticos con filtros y galería  
- agencias.html → Perfiles de agencias  
- precios.html → Comparación de precios  
- blog.html → Blog de destinos estilo revista  
- contacto.html → Formulario de contacto avanzado

## 🎨 Tecnologías Utilizadas

- HTML5  
- CSS3  
- Flexbox  
- CSS Grid  
- Animaciones con @keyframes  
- Variables CSS (Custom Properties)  

---

## ✨ Funcionalidades Principales

- Sección Hero con video de fondo  
- Mega-menú de navegación  
- Destinos destacados con tarjetas interactivas  
- Sistema de filtrado utilizando solo CSS  
- Galería tipo masonry con CSS Grid  
- Carrusel de testimonios sin JavaScript  
- Tabla de precios responsiva  
- Sistema de rating con estrellas (CSS-only)  
- Formulario con validación en tiempo real  
- Modo oscuro (Dark Mode)  
- Animaciones y micro-interacciones  

---

## 📱 Diseño Responsivo

El sitio está optimizado para distintos dispositivos:

- Mobile  
- Tablet  
- Desktop  

Se utilizan media queries, Flexbox y CSS Grid para adaptar el contenido.

---

## ♿ Accesibilidad

Se aplican buenas prácticas de accesibilidad:

- Contraste adecuado de colores  
- Navegación por teclado  
- Estados :focus visibles  
- Uso de etiquetas semánticas  
- Atributos ARIA en elementos interactivos  

---

## ⚙️ Consideraciones Técnicas

- No se utilizan frameworks (Angular, React, etc.)  
- No se utilizan componentes dinámicos  
- El header y footer se repiten manualmente en cada página  
- La reutilización se realiza mediante CSS
